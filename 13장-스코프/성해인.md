# 13. 스코프

## 13.1 스코프란?

> 스코프(scope, 유효범위)
> ***
> * **모든** 프로그래밍 언어의 기본 중요 개념
> * 변수, 함수와 깊은 관련 있음
> * 자바스크립트 스코프는 다른 언어의 스코프와 구별되는 특징 있음
>   * var 키워드 선언 변수의 스코프, let/const 키워드 선언 변수 스코프는 다르게 동작

#### 예제 13-01: 이미 경험한 스코프 예(1)<br>함수의 매개변수 스코프는 함수 몸체 내부로 한정<br>= 함수 매개변수는 함수 몸체 내부에서만 참조 가능
```Javascript
function add(x, y) { // 함수 매개변수 x, y
    console.log(x, y); // 매개변수 x, y 스코프는 함수 몸체 내부(외부에서는 참조X) // add(2, 5); // 2, 5
    return x + y;
}

add(2, 5);

console.log(x, y); // ReferenceError: x is not defined // 함수 내부에서만 참조 가능한 함수 매개변수 x, y // x가 먼저 쓰여서 x 변수에 대해 error 알림 나타남
```

> 모든 식별자(변수/함수/클래스 이름)는(은) 자신이 선언된 위치에 의해 다른 코드가 식별자 자신을 참조할 수 있는 유효 범위(스코프)가 결정된다.
> ***
> 스코프란? 식별자가 유효한 범위
> 또는? 자바스크립트 엔진이 식별자를 검색할 때 사용하는 규칙
> * 자바스크립트 엔진은 코드를 실행할 때 코드의 문맥(content)을 고려함
> 또는? 네임스페이스라고도 일컬을 수 있다.


* 변수는 (1)코드의 가장 바깥 영역뿐만 아니라 (2)코드 블록이나 (3) 함수 몸체 내에서도 선언 가능
  * 코드 블록/함수는 중첩 가능

#### 예제 13-02
```Javascript
var var1 = 1; // (1) 코드의 가장 바깥 영역에서 선언한 변수

if (true) {
    var var2 = 2; // (2) 코드 블록 내에서 선언한 변수
    if (true) {
        var var3 = 3; // (2*) 중첩된 코드 블록 내에서 선언한 변수
    }
}

function foo() {
    var var4 = 4; // (3) 함수 내에서 선언한 변수
    function bar() {
        var var5 = 5; // (3*) 중첩된 함수 내에서 선언한 변수
    }
}

console.log(var1); // 1 // 코드의 가장 바깥 영역 -> 참조 ok
console.log(var2); // 2 // true 조건 코드 블록 내부 -> ok
console.log(var3); // 3 // true 조건 중첩 코드 블록 내부 -> ok

// 함수 내에서 선언된 변수는 참조하지 못하는 현상
console.log(var4); // ReferenceError: var4 is not defined
console.log(var5); // ReferenceError: var5 is not defined
```

#### 예제 13-03: 식별자 결정(identifer resolution)

> 식별자가 지켜야 할 조건
> ***
> 1. 식별자는 유일해야 한다(unique) -> 식별자인 변수 이름은 중복 불가
> 2. 하나의 값은 유일한 식별자에 연결(name binding)되어야 한다.


> 식별자 결정(identifier resolution)
> ***
> 이름이 같은 2개 이상의 변수 중에서 어떤 변수를 참조해야 할지 결정하는 것

> 코드의 문맥과 환경
> ***
> * 렉시컬 환경(lexical environment): 코드의 문맥(content)
>   * 즉, 코드가 어디에서 실행되며 주변에 어떤 코드가 있는지에 관해
> * 실행 컨텍스트(execution content): 코드의 문맥을 구현한 것, 모든 코드는 실행 컨텍스트에서 평가되고 실행
>   * 스코프는 실행 컨텍스트와 깊은 관련이 있음


```Javascript
var x = 'global'; // 어디서든 참조 가능

function foo() {
    // var x = 'global'; 의 x와 이름은 동일한 식별자지만 스코프가 다른 별개의 변수
    var x = 'local'; // 식별자 결정(identifier resolution) #1 // foo 함수 내부 선언 -> foo 함수 내부에서만 참조 가능
    console.log(x); // (1)번 
}

foo();

console.log(x); // (2번)
```


> var 키워드 선언 변수의 중복 선언
> ***
> var 키워드 선언 변수는 같은 스코프 내에서 중복 선언 허용
> 단, 이로 인해 의도치 않게 변수값이 재할당되어 변경되는 부작용이 나타날 수 있음

#### 예제 13-04: var 키워드 선언 변수의 중복 선언
```Javascript
function foo() {
    var x = 1;
    var x = 2; // var 키워드 선언 변수는 동일 스코프 내에서 중복 선언 허용 // 이 시점에서 위의 var x = 1;는 없는 것으로 취급됨
    console.log(x); // 2
}
foo();
```

#### 예제 13-05: let/const 키워드 선언 변수의 중복 선언 불허
```Javascript
function bar() {
    let x = 1;
    let x = 2; // SyntaxError: Identifier 'x' has already been declared
}
bar();
```

## 13.2 스코프의 종류

> 코드의 분류(전역/지역)
> ***
> 전역 코드(global)
> 지역 코드(local)
> ***
> * 변수는 선언된 위치에 따라 스코프(유효 범위)가 결정된다.
>   * 전역: 전역 스코프를 갖는 전역 변수(global variable)
>   * 지역: 지역 스코프를 갖는 지역 변수(local variable)
> ***
> * 전역(전역 스코프, global scope): 코드의 가장 바깥 영역. 어디서든지 참조 가능
> * 지역(지역 스코프, local scope): 함수 몸체 내부. 자신의 지역 스코프와 하위 지역 스코프에서 유효


| 구분 |      설명      |  스코프   |  변수   |
|:--:|:------------:|:------:|:-----:|
| 전역 | 코드의 가장 바깥 영역 | 전역 스코프 | 전역 변수 |
| 지역 |   함수 몸체 내부   | 지역 스코프 | 지역 변수 |

* 스코프를 벗어난 변수를 참조하려고 하면 참조 에러(ReferenceError)가 발생한다.
* 자바스크립트 엔진은 스코프 체인을 통해 참조할 변수를 검색(identifier resolution)한다.

### 13.2.1 전역과 전역 스코프, 13.2.2 지역과 지역 스코프

#### 그림 13-2: 전역/지역 스코프
```Javascript
var x = "global x"; // 전역은 어디서든지 다 통함
var y = "global y";

function outer() { // outer 함수 내부가 outer 함수의 지역 스코프
  var z = "outer's local z";
  
  console.log(x); // global x
  console.log(y); // global y
  console.log(z); // outer's local z
  
  function inner() { // inner 함수 내부가 inner 함수의 지역 스코프
      var x = "inner's local x";
      
      console.log(x); // (4) // inner's local x // 자바스크립트 엔진: 스코프 체인 통해 참조할 변수 검색 -> inner함수 내부에서 선언된 x 변수 value 참조
      console.log(y); // (5) // global y
      console.log(z); // (6) // outer's local z
  }
  
  inner();
}
outer();

console.log(x); // global x
console.log(z); // ReferenceError: z is not defined
```

## 13.3 스코프 체인

> 함수의 중첩; 함수 몸체 내부에서 함수가 정의된 것
> * 중첩 함수(nested function): 함수 몸체 내부에서 정의한 함수
> * 외부 함수(outer function): 중첩 함수를 포함하는 함수
> ***
> 스코프는 함수의 중첩에 의해 계층적 구조를 가짐(상위/하위 스코프)
> * (예) 위의 그림 13-2에서 inner 함수는 outer 함수의 중첩 함수이며,<br>outer함수의 지역 스코프는 inner함수의 지역 스코프의 상위 스코프다.<br>outer함수 지역 스코프의 상위 스코프는 전역 스코프다.

> 스코프 체인(scope chain): 스코프가 계층적으로 연결된 것
> * 스코프 체인은 물리적인 실체로 존재함
>   1. 자바스크립트 엔진은 코드 실행 전 렉시컬 환경(Lexical Environment, 자료구조의 일종)을 실제로 생성
>   2. 변수 선언 실행 후, 변수 식별자가 렉시컬 환경에 키(key)로 등록
>   3. 변수 할당이 일어나면 해당 자료구조의 해당 식별자에 해당하는 값을 변경함
>   * 변수의 검색도 해당 자료구조 상에서 이루어짐
> ***
> 자바스크립트 엔진의 변수 검색(Identifer Resolution):
> 변수 참조시 자바스크립트 엔진은, (0) 스코프 체인을 통해 
> <br>(1) 변수를 참조하는 코드의 스코프에서 시작하여 
> <br>(2) 상위 스코프 방향으로 이동하며 
> <br>(3) 선언된 변수를 검색함.
> 이를 통해 상위 스코프에서 선언한 변수를 하위 스코프에서 참조 가능


### 13.3.1 스코프 체인에 의한 변수 검색

> 상속(inheritance, 부자관계로 구성)과 유사: 하위에서는 상위를 참조할 수 있지만, 상위에서 하위를 참조할 수 없다는 점
> * 상속을 통해 부모의 자산을 자식이 자유롭게 사용할 수는 있지만, 자식의 자산을 부모가 사용할 수는 없음
> ***
> 자바스크립트 엔진은 스코프 체인을 따라 
> <br>변수를 참조하는 코드의 스코프에서 시작해서
> <br>상위 스코프 방향으로 이동하며
> <br>선언된 변수를 검색한다
> * 절대 하위 스코프로 내려가면서 식별자 검색X
> ***
> <b>상위 스코프에서 유효한 변수는 하위 스코프에서 자유롭게 참조할 수 있지만, <br>하위 스코프에서 유효한 변수를 상위 스코프에서 참조X</b>

* 그림 13-2에서 일어나는 자바스크립트 엔진 스코프 체인 > 변수 검색 방법 분석
1. (4)
   1. x 변수 참조 스코프 inner함수 지역 스코프 → x 변수 선언 여부 검색
   2. 선언된 x 변수 존재
   3. 검색된 변수 참조, 검색 종료
2. (5)
   1. y 변수 참조 스코프 inner함수 지역 스코프 → y 변수 선언 여부 검색
   2. 선언된 y 변수 부존재
   3. 상위 스코프인 outer함수 지역 스코프 → y 변수 선언 여부 검색
   4. 선언된 y 변수 부존재
   5. 상위 스코프인 전역 스코프 → y 변수 선언 여부 검색
   6. 선언된 y 변수 존재
   7. 검색된 변수 참조, 검색 종료
3. (6) 
   1. z 변수 참조 스코프 inner함수 지역 스코프 → z 변수 선언 검색
   2. 선언된 z 변수 부존재
   3. 상위 스코프인 outer 함수 지역 스코프 → z 변수 선언 검색
   4. 선언된 z 변수 존재
   5. 검색된 변수 참조, 검색 종료
   


### 13.3.2 스코프 체인에 의한 함수 검색

#### 예제 13-06

> 함수 선언문으로 함수 정의시, 런타임 이전에 함수 객체 먼저 생성
> * 자바스크립트 엔진은 함수 이름과 동일한 이름의 식별자를 암묵적으로 선언한 후, 생성된 함수 객체 할당

```Javascript
function foo() {

}

    }
    foo(); // (1)에서 foo 함수 호출 → Javascript Engine은 함수를 호출하기 위해 먼저 함수 식별자 foo를 검색함
  // 스코프는 사실 식별자를 검색하는 규칙(변수를 검색하는 로직이나 함수를 검색하는 로직이나 별 다를 바 없음)
  // local foo 를 참조할 것
}

bar();
```


## 13.4 함수 레벨 스코프

> 지역: 함수 몸체 내부, 지역 스코프 생성
> → 코드 블록이 아닌 함수만 지역 스코프가 생성

> 블록 레벨 스코프(block level scope): 함수 몸체만이 아니라 모든 코드 블록이 지역 스코프 생성
>   * C, Java 등 대부분의 프로그래밍 언어가 여기에 해당
>   * (Javascript ES6) let, const 변수는 블록 레벨 스코프 지원
> 함수 레벨 스코프(function level scope)
>   * (Javascript) var 키워드로 선언된 변수: 오로지 **함수**의 코드 블록(함수 몸체)만을 지역 스코프로 인정함



#### 예제 13-07
```Javascript
var x = 1;

if (true) {
    var x = 10; // var 키워드 선언 변수: 함수의 코드 블록(함수 몸체)만을 지역 스코프로 인정함
  // '함수 밖'에서 var 키워드로 선언된 변수는 코드 블록 내에서 선언되었어도 모두 전역 변수로 취급
  // if문 안의 x변수는 전역 변수로 취급, 바깥에서 1로 선언됐던 게 10으로 중복 선언됨(의도치 않게? 변수 값 변경 부작용 발생 가능)
}

console.log(x); // 10
```

#### 예제 13-08
```Javascript
var i = 10;

// for문은 함수 취급x & var 변수니 i는 전역 변수 취급 // 이미 선언된 전역 변수가 있으므로 중복 선언된 꼴
for (var i = 0; i < 5; i++) {
    console.log(i); // 0 1 2 3 4
}

console.log(i); // 전역에 선언됐던 초기 10 value가 변경 // 5
```


## 13.5 렉시컬 스코프


#### 예제 13-09: 다음 코드를 실행했을 때 어떤 결과가 나올까?

> Q. bar 함수의 상위 스코프는 함수가 어디에서 **호출**되었는지에 따라 달라질까,<br>아니면 어디에서 **정의**되었는지에 따라 달라질까?
> ***
> 1. 동적 스코프(dynamic scope): 함수 호출 시점에 함수 상위 스코프 결정
> 2. 정적 스코프(static scope, lexical scope): **함수 정의 시점**에 함수 상위 스코프 결정
>    * 자바스크립트 비롯 대부분의 프로그래밍 언어는 정적 스코프(렉시컬 스코프) 방식을 따름
> ***
> ***자바스크립트는 렉시컬 스코프를 따르므로, 함수 정의 시점에 따라 상위 스코프 결정***
> 함수 정의(함수 선언문/함수 표현식)가 실행되어 생성된 함수 객체는 생성시 결정된 상위 스코프를 기억함
> * 함수 호출시 함수의 상위 스코프를 참조할 필요가 있음


```Javascript
var x = 1;

function foo() { // 전역에서 생성된 함수
    var x = 10;
    bar();
}

function bar() { // 전역에서 생성된 함수
    // 함수 선언문으로 정의된 bar 함수는 전역 코드 실행 전에 먼저 평가, 함수 객체 생성, 이때 전역 스코프 기억함
    console.log(x);
}

foo(); // foo 함수 내부에서 bar 함수를 실행시켜도 bar 함수는 생성시점 상위 스코프를 기억 // 상위 스코프 = 전역이므로 // 1
bar(); // 상위 스코프 = 전역이므로 // 1
```
